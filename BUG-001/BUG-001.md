## BUG-001: AGORA 1.0.0: User Signup Password Mismatch: Silent Failure Password Mismatch
 
**[Summary]** During the user registration process, entering mismatched passwords in the "Password" and "Confirm Password" fields results in a silent failure; the form does not submit, but no error message or validation warning is displayed to the user.

**[Precondition]**
* Software version: 1.0.0
* Software configuration: Web application signup page
* Hardware specifications: Standard Web Browser (Chrome, Firefox, Edge, Safari)
* Network configuration: Standard Internet Connection
 
**[Steps to reproduce]**
1. Navigate to the AGORA user signup page.
2. Fill out all required fields (e.g., Name, Email) with valid information.
3. Enter a valid password in the "Password" field (e.g., Password123!).
4. Enter a different password in the "Confirm Password" field (e.g., Password456!).
5. Click the "Sign Up" or "Submit" button.
6. Observe that the user remains on the signup page with no visual indicators, error messages, or validation alerts to signal a password mismatch.

**[Actual results]** The registration form is not submitted, and the signup process does not advance. However, the interface remains completely unchanged—there is no text alert, red highlight, or toast notification informing the user that the passwords do not match.

**[Expected results]** The form should block submission and immediately display a clear validation error message (e.g., "Passwords do not match") below the password fields to notify the user of the error.
