## BUG-002: AGORA 1.0.0: Create Election Session Invalid Date Range: Can Set Election in the Past
 
**[Summary]** The system allows a user to create an election session with an invalid date range that was set in the past, failing to validate the input against the current calendar date.

**[Precondition]**
* Software version: 1.0.0
* Software configuration: Web application election session creation form
* Hardware specifications: Standard Web Browser
* Network configuration: Standard Internet Connection

**[Steps to reproduce]**
1. Log in to the application with an account that has election creation privileges.
2. Navigate to the "Create Election Session" page.
3. Fill out all required fields with valid details.
4. Locate the election date settings and input a start and end date that has already passed (e.g., a date from last month or last year).
5. Click the "Create" or "Submit" button.
6. Observe the validation response and system behavior.

**[Actual results]** The system successfully accepts the past dates without validation errors, processes the form creation, and saves the session with an outdated chronological timeline.

**[Expected results]** The system should block form submission and display a validation error message (e.g., "Election date cannot be in the past" or "Please select a valid future date range").
