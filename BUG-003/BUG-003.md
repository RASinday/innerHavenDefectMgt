## BUG-003: AGORA 1.0.0: Create/Edit Organization Duplicate Name: Duplicate Organization Name Allowed
 
**[Summary]** The system allows duplicate organization names to be saved during both creation and editing, failing to flag or block names that match an existing organization on the platform.

**[Precondition]**
* Software version: 1.0.0
* Software configuration: Web application organization management dashboard
* Hardware specifications: Standard Web Browser
* Network configuration: Standard Internet Connection
* Setup State: An organization with a specific name (e.g., "Alpha Corp") is already visible/created on the platform.

**[Steps to reproduce]**
1. Log in to the application with an account that has organization administrative privileges.
2. **Scenario A (Creation):** Navigate to the "Create Organization" page and enter the existing name ("Alpha Corp") into the organization name field. Click submit.
3. **Scenario B (Editing):** Navigate to a completely different organization, click "Edit", change its name field to match the existing organization ("Alpha Corp"), and click save.
4. Observe the user interface behavior and validation responses for both scenarios.

**[Actual results]** The system successfully processes both operations without any validation errors or warnings, resulting in multiple distinct organization profiles listing the exact same name string.

**[Expected results]** The system should catch duplicate names during both creation and modification. Form submission should be blocked, displaying a clear validation warning (e.g., "An organization with this name already exists. Please choose a unique name.").
